# DHAROHAR - Development Walkthrough

## Latest Project Progress (Phases G-L)

DHAROHAR is an indigenous cultural governance platform. This walkthrough summarizes the latest implementations including the licensing framework, mobile responsiveness, and end-to-end media archival workflows.

The entire platform is now fully responsive for mobile (portrait/landscape) and tablet devices.
- **Navigation**: Slide-in **Hamburger Drawer** for mobile viewports with a sticky top bar.
- **Layouts**: All grids, forms, and selection cards stack vertically on small screens.
- **Touch Targets**: Minimum **44px height** for all buttons.

### 2. End-to-End Media Visibility (Phase L)
Resolved the core workflow gap that prevented media files from being seen across roles:
- **Backend Synthesis**: `assetService` now computes dynamic `mediaUrl` strings for all queries.
- **License Population**: Fixed the `populate()` chain to include media fields for admins and reviewers.
- **Public Streaming**: Relaxed auth guards for GET requests to allow public media previews (30s).
- **GridFS Integration**: Fixed the upload form to store media blobs in MongoDB before creating metadata records.

### 3. Role-Based Audio/Video Playback (Phase J)
Universal `RoleMediaPlayer` component with two modes:
- **Full**: Unrestricted playback for Community (own assets), Reviewers, and Admins.
- **Preview**: 30-second capped playback for public users with a "License Required" CTA.

| Context | Role | Playback Mode |
|---|---|---|
| My Submissions | Community | Full |
| Review Queue | Reviewer | Full |
| License Requests | Admin | Full |
| Cultural Explorer | Public | Preview (30s) |
| My Licenses | General | Full (after approval) |
